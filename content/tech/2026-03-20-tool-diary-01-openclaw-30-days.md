# 工具人日記 #1：用 OpenClaw 30 天後，我的工作流徹底變了

> 系列說明：「工具人日記」每週一篇，記錄我實際使用各種 AI 工具的真實體驗——不是業配，不是教學，是踩坑日記。

## 先說結論

OpenClaw 不是「又一個 AI 聊天工具」。它是一個**住在你電腦裡的 AI 助手**，能讀你的檔案、管你的 GitHub、幫你排程任務、甚至在你睡覺時自動工作。

用了 30 天後，我的工作流從這樣：

```
06:00 起床 → 手動查新聞 → 寫摘要 → 找圖片 → 發文 → git push
（耗時：2-3 小時）
```

變成這樣：

```
06:00 Cron 自動觸發 → AI 搜新聞 → 自動寫文 → 自動配圖 → 自動 push
06:30 我起床看 Telegram 通知，確認結果
（我的耗時：5 分鐘）
```

這不是誇張。這是我昨天的真實情況。

## 30 天踩坑紀錄

### 第 1 週：興奮期

剛裝好 OpenClaw，感覺像擁有了 Jarvis。能直接在 Telegram 上跟它對話、叫它讀檔案、搜資料、寫程式。第一天就讓它幫我整理了 100 多篇 blog 文章的 metadata。

**踩坑**：一開始太依賴它的記憶，結果 session 一清就什麼都忘了。後來才搞懂要用 `memory_store` 存重要決策，用 `memory/YYYY-MM-DD.md` 寫每日摘要。

### 第 2 週：自動化期

開始設定 Cron 排程。讓它每天早上自動掃 Google Trends + Hacker News，寫成中文 blog 文章，配上 AI 生成圖片，然後 git push。

**踩坑**：Meta AI 的 MCP server 每次呼叫都會 spawn 新進程但不清理，一天下來堆了 30 個殭屍進程。解法：寫了一個 cleanup 腳本，排進每日 Cron。

### 第 3 週：代理協作期

嘗試用 `sessions_spawn` 讓多個 AI 代理並行工作。一個負責找缺圖文章，一個負責產圖，一個負責 git 同步。效率翻倍，但協調成本也增加了。

**踩坑**：子代理不繼承父代理的所有工具權限。例如 isolated session 的代理不能呼叫 browser 工具。需要在設計時就想清楚工具分配。

### 第 4 週：穩定運營期

系統趨於穩定。5 個 Cron 任務每天自動執行，blog 文章穩定產出，GitHub 自動同步。我的角色從「操作者」變成「監督者」。

**踩坑**：sync-skills 的 rsync 腳本把目標 repo 的 `.git` 目錄也覆蓋了，導致 git push 連續失敗 7 次。教訓：rsync 永遠要加 `--exclude='.git'`。

## 我的真實評價

**優點：**
- 🟢 真正的「AI 助手」，不只是聊天機器人
- 🟢 Cron + 子代理 = 強大的自動化能力
- 🟢 記憶系統讓它越用越懂你
- 🟢 開源免費，沒有月費負擔

**缺點：**
- 🔴 學習曲線陡峭，設定需要技術背景
- 🔴 記憶系統需要手動維護，不是完全自動
- 🔴 MCP 工具（如 Meta AI）穩定性不夠
- 🔴 Context window 有限，長對話會被壓縮丟失細節

**適合誰：**
- ✅ 有技術背景的個人開發者
- ✅ 需要管理多個專案的 solo founder
- ✅ 想要自動化重複工作的內容創作者
- ❌ 不適合完全不懂 CLI 的人
- ❌ 不適合需要 100% 準確率的場景

## FAQ

**Q: OpenClaw 是免費的嗎？**
A: OpenClaw 本身開源免費，但需要自己的 AI API key（如 Anthropic Claude），API 費用另計。

**Q: 需要什麼技術背景？**
A: 至少要會用 CLI、理解 Git 基本操作、能看懂 JSON 設定檔。

**Q: 和 ChatGPT 有什麼不同？**
A: ChatGPT 是對話工具，OpenClaw 是 AI 代理平台——它能讀寫你的檔案、執行指令、排程任務、呼叫外部 API。

**Q: 資料安全嗎？**
A: 所有資料都在你自己的機器上，OpenClaw 不上傳任何東西到雲端（除了你自己設定的 API 呼叫）。

**Q: 一個月的 API 成本大約多少？**
A: 看使用量。我大量使用的情況下，Claude API 月費約 $30-50 USD。用免費模型（如 Groq）可以降到接近零。
